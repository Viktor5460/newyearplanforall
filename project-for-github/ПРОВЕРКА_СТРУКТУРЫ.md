# 📋 Проверка структуры репозитория

## ✅ Правильная структура для Render:

```
ваш-репозиторий/
├── app.py                 ← ОБЯЗАТЕЛЬНО в корне!
├── schedule_data.py       ← ОБЯЗАТЕЛЬНО в корне!
├── requirements.txt       ← ОБЯЗАТЕЛЬНО в корне!
├── Procfile              ← ОБЯЗАТЕЛЬНО в корне!
├── templates/
│   ├── mailbox.html
│   └── letters.html
└── static/
    ├── css/
    │   └── letters.css
    └── js/
        └── letters.js
```

## ❌ НЕПРАВИЛЬНО (файлы в подпапке):

```
ваш-репозиторий/
└── project-for-github/    ← НЕПРАВИЛЬНО!
    ├── app.py
    ├── requirements.txt
    └── ...
```

## 🔍 Как проверить на GitHub:

1. Откройте: `https://github.com/Viktor5460/NewYearPlan`
2. Файлы должны быть видны СРАЗУ (не нужно заходить в подпапку)
3. Должны быть видны:
   - app.py
   - requirements.txt
   - schedule_data.py
   - Procfile
   - папка templates/
   - папка static/

## ✅ Если структура правильная:

Файлы в корне → Render найдет app.py → всё заработает!

## ❌ Если файлы в подпапке:

Вариант 1: Переместить файлы в корень (рекомендуется)  
Вариант 2: В Render указать Root Directory: `project-for-github`
